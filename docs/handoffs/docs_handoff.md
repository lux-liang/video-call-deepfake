# Docs Handoff

## 负责范围

- 维护 README、架构文档、demo 脚本、验收清单和术语一致性。
- 对齐 backend / pipeline / frontend 的字段命名和叙述口径。

## 当前输入

- Phase 1 已有初版文档与 handoff 合同。

## 交付要求

- 所有核心文档必须反复强调 AI workflow 定位。
- README 中必须显式覆盖 Harness Engineering 三件套。
- demo 脚本需能支撑课程展示与 PR 说明。
- README 必须显式覆盖课程要求中的痛点定义、非聊天网页、GitHub 交付和 3 分钟 demo。
- 验收清单必须可逐项检查，不能只写成抽象口号。
- 若其他分支尚未完成，文档必须写出待补位说明和验收占位。

## 固定术语

- `participant`
- `event`
- `response`
- `suspicious segment`
- `tool logs`
- `report summary`

除非合同变更，否则不要在 README、demo、handoff 中切换为别名。

## 非目标

- 不把文档写成泛泛项目介绍。
- 不遗漏 fallback、限制和未来边界。
